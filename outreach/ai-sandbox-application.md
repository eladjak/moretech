### 2. תיאור המוצר

**EduTech** היא פלטפורמת AI מקצועית בעברית עבור מורים ישראליים, שנועדה לחסוך זמן, לשפר את איכות ההוראה ולהנגיש את טכנולוגיית הבינה המלאכותית לכלל מערכת החינוך.

#### כלים עיקריים בפלטפורמה:

| כלי | תיאור |
|-----|--------|
| **מתכנן שיעורים** | יצירת תכניות שיעור מותאמות לתוכנית הלימודים הישראלית תוך שניות |
| **מחולל בחינות ומבדקים** | יצירת שאלות בפורמט בגרות, בחינות, עבודות מסכמות |
| **התאמת לומדים** | התאמת חומרי לימוד לתלמידים עם צרכים מגוונים (שונות לומדים) |
| **מכתבי הורים** | כתיבה אוטומטית של עדכונים, הזמנות וחוזרים |
| **AI-Assistant** | עוזר AI אישי המבין הקשר חינוכי ישראלי, בעברית מלאה |
| **בנק משאבים** | ספרייה של פעילויות, דפי עבודה ומשימות מוכנות |

#### בידול טכנולוגי:
- **Hebrew-native בלבד** — המוצר בנוי מהיסוד בעברית, לא תרגום ממוצר אנגלי
- **RTL-first** — ממשק, מסמכים ופלטים כולם מותאמי ימין-לשמאל
- **מותאם לתוכנית הלימודים הישראלית** — פורמט בגרות, מקצועות, שכבות גיל מוגדרות
- **AI Hybrid** — שילוב Claude API + Gemini API לאיכות פלט מיטבית בעברית
- **Stack טכנולוגי:** Next.js + Supabase + Claude API + Gemini API + next-intl

#### השראה ומיצוב שוק:
המוצר מונע מהצלחת **MagicSchool** (ארה"ב) — $45 מיליון הכנסות שנתיות, 6 מיליון מורים — עם אפס נוכחות בעברית. EduTech ממלאת את החלל הזה בשוק הישראלי עם פתרון מותאם תרבות ושפה.

### 3. הבעיה שנפתרת

#### 3.1 בעיית השפה — המחסום הגדול ביותר

**כיום, אין אף כלי AI ייעודי בעברית למורים ישראליים.**

כל הכלים הקיימים (MagicSchool, Khanmigo, Diffit, Magic Write) הם באנגלית בלבד. מורים ישראליים ש מנסים להשתמש בהם נתקלים ב:
- פלט באנגלית שדורש תרגום ועריכה ידנית
- היעדר הבנת הקשר חינוכי ישראלי
- ממשקי LTR (שמאל-לימין) שמקשים על עבודה עם תוכן עברי
- חוסר התאמה לבגרות, לתוכנית המשרד, למועדי שנת הלימודים הישראלית

#### 3.2 משבר הזמן של המורים

| נתון | מקור |
|------|------|
| **15+ שעות שבועיות** שמורים מבלים על משימות אדמיניסטרטיביות | סקרי משרד החינוך |
| **76% ממורים** חסרי הכשרה בסיסית לשימוש בAI | סקר מורים 2025 |
| **60% ממורים** מצהירים שרוצים להשתמש ב-AI אך אינם יודעים איך | נתוני OECD ישראל |
| **40%** מהמורים בישראל מדווחים על שחיקה ברמה גבוהה | דוח משרד החינוך 2024 |

#### 3.3 כשל יישום ב"שנת ה-AI" הממשלתית

הממשלה הכריזה על "שנת AI בחינוך" ושמה תקציב של **1.485 מיליארד ש"ח** לקידום STEM. אולם בפועל:
- בתי הספר מקבלים הנחיות ל"שלב AI בהוראה" ללא כלים מעשיים בעברית
- מורים מנסים להשתמש ב-ChatGPT הכללי ומקבלים תוצאות גנריות ולא מותאמות
- הכשרות מורים קיימות מלמדות תיאוריה אך לא כלים ספציפיים לעבודה יומיומית
- אין מדידה מסודרת של השפעת AI על תוצאות לימודיות בישראל

**EduTech מספקת את הפתרון המעשי שחסר: כלי, הדרכה, ומדידה — בעברית.**

### 4. חדשנות ויתרון תחרותי

#### 4.1 ניתוח שוק — למה אין מתחרים ישירים?

| מוצר | שפה | מותאם לישראל | AI-Native | חינוך ספציפי |
|------|-----|--------------|-----------|--------------|
| MagicSchool | אנגלית בלבד | לא | כן | כן |
| Khanmigo | אנגלית | לא | כן | כן |
| ChatGPT | רב-לשוני, לא ייעודי | לא | כן | לא |
| Google Bard/Gemini | רב-לשוני, לא ייעודי | לא | כן | לא |
| כלים ישראליים קיימים | עברית | חלקית | לא | חלקית |
| **EduTech** | **עברית-native** | **כן** | **כן** | **כן** |

#### 4.2 מודל עסקי חדשני: PLG + שטח

**PLG (Product-Led Growth):** המורה מגלה את הכלי בחינם, משתמש, מתלהב, ומשכנע את המנהל לרכוש רישיון לבית הספר.

**הרחבת שטח:** בניגוד ל-MagicSchool שהיא SaaS טהורה, EduTech משלבת:
- פלטפורמת SaaS
- סדנאות הטמעה פיזיות (MoreTech — המותג לסדנאות)
- ליווי מתמשך

**התוצאה:** שיעור נטישה (Churn) מתחת ל-10% (לעומת 30-40% ב-SaaS חינוכי ממוצע).

#### 4.3 חסם כניסה

**Hebrew-native AI requires significant investment.** Building authentic Hebrew educational AI — not just UI translation — demands:
- Fine-tuning על תוכן חינוכי ישראלי
- בניית פרומפטים המבינים הקשר בגרות ומקצועות ישראליים
- קורפוס של חומרי לימוד ישראליים
- הכרת מערכת הציונים, לוח השנה, ואופי התוכנית הלאומית

זהו יתרון שמצטבר עם הזמן ויקשה על מתחרים גדולים להעתיק בקלות.

### 5. הפיילוט המוצע

#### 5.1 מטרת הפיילוט

להוכיח שכלי AI בעברית מובנה מגדיל משמעותית את פרודוקטיביות המורים ואיכות ההוראה, ולספק נתוני מחקר שיאפשרו הרחבה לכלל מערכת החינוך הישראלית.

#### 5.2 פרמטרים

| פרמטר | פרטים |
|--------|--------|
| **משך** | 8 שבועות |
| **מורים** | 50 מורים |
| **בתי ספר** | 2-3 בתי ספר |
| **שכבות גיל** | ז'-יב' (חטיבה + תיכון) |
| **מקצועות** | אנגלית, היסטוריה, אזרחות, מדעים (מגוון) |
| **רשת מועדפת** | AMIT (63 בתי ספר) או ORT (237 בתי ספר) |
| **מיקום גאוגרפי** | מרכז הארץ (נגישות למעקב) |

#### 5.3 שלבי הפיילוט

**שבוע 1-2: הכנה ואונבורדינג**
- הגדרת חשבונות למורים משתתפים
- סדנת הדרכה (3 שעות, פיזית) לכל בית ספר
- Baseline measurement: שעות עבודה שבועיות, שביעות רצון, איכות שיעורים

**שבוע 3-6: שימוש עצמאי מלווה**
- מורים משתמשים בפלטפורמה ב-5 תרחישי שימוש ליבה
- Check-in שבועי קצר (15 דקות) לאיסוף feedback
- תמיכה טכנית זמינה

**שבוע 7-8: הערכה ומסקנות**
- מדידת תוצאות מול Baseline
- ראיונות עומק עם 10 מורים נבחרים
- ניתוח נתוני שימוש בפלטפורמה
- הכנת דוח מחקרי

#### 5.4 מדדי הצלחה (KPIs)

| מדד | יעד |
|-----|-----|
| **חיסכון בזמן** | 5+ שעות שבועיות למורה (מ-15 ל-10) |
| **שביעות רצון** | 80%+ דיווח "כלי שימושי מאוד" |
| **שיעור שימוש פעיל** | 80%+ מהמורים משתמשים לפחות 3 פעמים שבועיות |
| **שיפור איכות שיעורים** | 70%+ מדווחים על שיפור בתכנון השיעורים |
| **המלצה** | NPS > 50 |
| **שיעור המשך שימוש** | 90%+ אחרי תום הפיילוט |

#### 5.5 תקציב מבוקש מה-Sandbox

| סעיף | עלות משוערת (ש"ח) |
|------|-------------------|
| פיתוח MVP ל-Pilot (3 חודשים) | 120,000 |
| תשתיות ענן + API costs | 30,000 |
| סדנאות הכשרה (מרצה + חומרים) | 25,000 |
| מחקר ומדידה (מחקר מלווה) | 40,000 |
| ניהול פרויקט + קואורדינציה | 35,000 |
| **סה"כ מבוקש** | **250,000 ש"ח** |

*(טווח מלא אפשרי עד 500,000 ש"ח בהתאם להיקף הפיילוט המאושר)*

### 7. התאמה לקריטריוני Sandbox

#### 7.1 למה EduTech מתאימה ל-Sandbox?

ה-Sandbox נועד בדיוק לפרויקטים כמו EduTech:

**מוצר AI-Native:** הפלטפורמה מבוססת LLM לחלוטין — ה-AI הוא הליבה, לא תוספת. כל תכולה נוצרת ומועברת דרך מודלי AI.

**תחום עדיפות ממשלתי:** חינוך הוא אחד ממוקדי ה-Sandbox בשנת 2026. פיילוט זה תואם ישירות ליעדי "שנת ה-AI" ותקציב ה-STEM הלאומי.

**שאלות רגולטוריות אמיתיות:** EduTech מעלה שאלות רגולטוריות שהחקיקה הנוכחית לא מכסה כראוי:

| שאלה רגולטורית | רלוונטיות ל-Sandbox |
|----------------|---------------------|
| **פרטיות נתוני תלמידים קטינים** | GDPR-IL חלקי — נדרשת מסגרת ספציפית לנתוני ילדים |
| **הסכמה מדעת להורים** | טפסי הסכמה לשימוש ב-AI בכיתה — אין תקן ישראלי |
| **הטיה (Bias) ב-AI בעברית** | זיהוי ומניעת הטיה בתוכן חינוכי שנוצר על ידי LLM |
| **אחריות על תוכן שגוי** | מי אחראי אם AI מייצר תוכן לימודי שגוי? |
| **אינטגרציה עם מערכות בית ספר** | ממשקי API למערכות Mashov, GEMS, ספריות דיגיטליות |
| **זכויות יוצרים** | AI שמבסס תוכן על ספרי לימוד מוגנים בזכויות |

**EduTech זקוקה ל-Sandbox** כדי:
1. לפעול עם נתוני תלמידים קטינים תוך בחינת מסגרת רגולטורית מתאימה
2. לבחון שאלות אחריות לגבי תוכן AI בחינוך פורמלי
3. לפתח תקן ישראלי לשימוש AI בבתי ספר שיוכל לשמש מדיניות לאומית

#### 7.2 הגנת פרטיות — עקרונות הפיילוט

- **אנונימיזציה מלאה:** כל נתוני תלמידים יאוחסנו ויעובדו ללא שמות מזהים
- **הסכמה מפורשת:** טפסי הסכמה להורים + מורים לפני כל שימוש
- **אחסון בישראל:** כל הנתונים על שרתים בישראל (AWS Tel Aviv / Azure Israel)
- **מדיניות מחיקה:** נתוני פיילוט יימחקו 6 חודשים לאחר סיום
- **ביקורת צד ג':** מינוי DPO (Data Protection Officer) לפיקוח
